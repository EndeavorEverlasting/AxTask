perf(api): tighten latency thresholds and add perf:api-replay CI gate

- GET /api/tasks p95 threshold dropped 800ms → 600ms and a new
  mutation_latency signal fires for slow POST/PUT/PATCH/DELETE so
  optimistic-UI regressions fail loudly instead of silently stretching
  the pending state.
- tools/perf/api-replay.mjs replays a JSON fixture of api_request events
  through the pure heuristics and exits non-zero when any critical
  signal fires. Baseline + regression fixtures ship in
  tools/perf/fixtures/ and the deploy contract now requires the
  perf:api-replay script to exist.
- Vitest server project picks up tools/**/*.test.{ts,tsx} so the replay
  script is itself covered end-to-end.
